# Matchday Generation Rules

Use these rules when generating, updating, or reviewing Caddyshack matchday content.

## Core conventions

- Say `Matchday`, not `Gameweek`.
- Show all odds as decimal odds.
- Use Ladbrokes as the bookmaker reference for option generation.

## Execution mode

- Default to a full rebuild for matchday generation and regeneration tasks.
- A full rebuild means all dependent parts are refreshed together, including:
  - proposals and bet lines
  - market snapshot selections
  - form rows and form matches for all included fixtures
  - timeline proposal marker totals
  - rationale text and sequencing notes
  - lessons-learned application across risk groups
  - required badge coverage for clubs used in the generated legs
- Only ask the user before full rebuild when the edit is unusually risky, destructive, or has significant non-obvious trade-offs.

## Risk groups

Always provide three acca groups:

- Defensive
- Neutral
- Aggressive

## Odds and segment rules

- Suggested bets should be checked against Ladbrokes odds or the latest local Ladbrokes snapshot.
- Every segment should include its decimal odds.
- Every segment should include a small day and kickoff strapline.
- Segments must appear in chronological order by date and kickoff time inside each acca option.

## Stake generation

- Do not hardcode stakes.
- Calculate stakes from the current pot, the original deposit baseline, and completed matchdays.
- Treat the Aggressive stake as the anchor.
- Keep the Aggressive stake near 50% of the pot, adjusting with progress and profit.
- Derive Defensive and Neutral stakes relative to the Aggressive stake using each acca's total decimal odds.

## Cashout guidance

Each acca should provide:

- Lower cashout
- Upper cashout
- No cashout value
- What to watch

Apply the following behaviour:

- Defensive:
  - protect cashout value earlier
  - favour steadier decision points
- Neutral:
  - balance protection and upside
  - reassess after early legs
- Aggressive:
  - prioritise return potential
  - only cash early when the risk picture materially worsens

## User state

- Do not make a default choice for the logged-in user.
- Persist the logged-in user's choice via local storage.
- If a local persisted choice exists, restore it on load.

## Ledger consistency

- Pot increases are positive entries with a green up-right arrow.
- Pot decreases are negative entries with a red down-right arrow.
- Stake placement must be represented as a negative ledger entry.
